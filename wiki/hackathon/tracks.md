# 5개 트랙 상세

> 출처: Devpost rules + resources 페이지. 각 트랙은 상호배타적이지 않으나 제출 시 1개 트랙 지정 필요.

## Track 1: MemoryAgent

**요구**: 영속 메모리를 갖고 경험을 자율 축적하며, 사용자 선호를 기억하고, 멀티턴·크로스세션에서 점점 더 정확한 의사결정을 하는 에이전트.

**심사 포커스 (공식 명시)**:
1. 효율적인 메모리 저장·검색
2. 오래된 정보의 적시 망각(timely forgetting)
3. 제한된 컨텍스트 윈도우 안에서 핵심 기억 회상

**공식 프로젝트 아이디어 예시**:
- 읽은 논문을 모두 기억하고 세션 간 종합하는 리서치 어시스턴트
- 대화 이력·선호를 재질문 없이 회상하는 고객지원 에이전트
- 공유한 모든 것을 인덱싱하고 관련 컨텍스트를 능동 제시하는 개인 지식 베이스

## Track 2: AI Showrunner

**요구**: Wan / HappyHorse 등 영상 생성 능력으로 숏드라마 제작 파이프라인 전체(각본→스토리보드→영상 생성→편집)를 자율 수행하는 에이전트.

**심사 포커스**: 내러티브 능력, 멀티모달 오케스트레이션, **제한된 토큰 예산 하 출력 품질 극대화**. 토큰 허용량이 가장 큰 트랙.

**아이디어 예시**: 자동 팟캐스트 파이프라인, 플레이어 선택 기반 내러티브 게임 엔진, 멀티에이전트 작가실.

## Track 3: Agent Society

**요구**: 서로 다른 능력의 에이전트들이 작업 분담·대화·협상으로 복잡한 과제를 함께 해결하는 멀티에이전트 협업 시스템.

**심사 포커스**:
1. 과제 분해와 역할 배정 방식
2. 의견 충돌·실행 충돌 해결 방식
3. **단일 에이전트 베이스라인 대비 측정 가능한 효율 향상** (정량 비교 필수)

**아이디어 예시**: 구매자·판매자 에이전트 협상 마켓, 멀티에이전트 토론 플랫폼, 협력적 문제해결 스웜.

## Track 4: Autopilot Agent

**요구**: 실제 비즈니스 워크플로우를 엔드투엔드로 자동화하는 에이전트. 시나리오 자유 (문의 메일→견적, 시스템 알람→자동 복구, 이력서 스크리닝→면접 일정 등).

**심사 포커스**:
1. 모호한 입력 처리 능력
2. 외부 도구 호출
3. 핵심 의사결정 지점의 **human-in-the-loop 체크포인트**
4. **토이 데모가 아닌 프로덕션 준비성** 강조

**아이디어 예시**: PR 분석→이슈 식별→수정 제안→검증하는 코드리뷰 파이프라인, 자율 데이터 분석가, 자가치유 배포 시스템.

## Track 5: EdgeAgent

**요구**: Qwen 기반 물리 디바이스(로봇·IoT·스마트 하드웨어). 엣지 센서로 인지, 클라우드 API/Skills로 추론, 로컬에서 실행.

**심사 포커스**:
1. 대역폭·지연 제약 하 엣지-클라우드 오케스트레이션
2. 프라이버시 인지 데이터 처리
3. 오프라인/약네트워크에서의 우아한 성능 저하(graceful degradation)

**주의**: 비공개 하드웨어 사용 시 주최 측이 물리적 접근을 요구할 수 있음.

## 트랙 간 비교 요약

| 트랙 | 핵심 역량 | 하드웨어 필요 | 정량 평가 요구 | 난이도 축 |
|---|---|---|---|---|
| 1 MemoryAgent | 메모리 저장·검색·망각 | ✗ | 암묵적 (회상 정확도) | 지식·컨텍스트 엔지니어링 |
| 2 AI Showrunner | 영상 생성·멀티모달 오케스트레이션 | ✗ | 출력 품질/토큰 | 크리에이티브 파이프라인 |
| 3 Agent Society | 멀티에이전트 협업·협상 | ✗ | **명시적** (vs 단일 에이전트) | 오케스트레이션·평가 설계 |
| 4 Autopilot | E2E 업무 자동화 + HITL | ✗ | 암묵적 (프로덕션성) | 도구 통합·신뢰성 |
| 5 EdgeAgent | 엣지-클라우드 하이브리드 | **✓** | 지연·오프라인 강건성 | 하드웨어·네트워크 |
